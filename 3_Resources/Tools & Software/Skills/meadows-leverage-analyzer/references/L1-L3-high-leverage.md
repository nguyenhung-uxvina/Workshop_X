# High Leverage Points (L1-L3) - Detailed Examples

## L1: Transcending Paradigms

**Definition**: Ability to keep paradigms flexible, question fundamental assumptions, not be rigidly attached to any worldview.

### Defense Manufacturing Example
**Scenario**: Vietnamese defense contractor struggling with "make vs buy" decisions

**Paradigm Identified**: "We must manufacture everything in-house to maintain security and self-reliance"

**Transcendence**: Question whether security comes from manufacturing or from integration capability. Could security be achieved through:
- Controlling critical integration knowledge
- Building best-in-class systems engineering
- Strategic partnerships with audited suppliers

**Impact**: Opens entirely new solution space beyond make/buy dichotomy

### Software Development Example
**Scenario**: Company culture of "real programmers write everything from scratch"

**Paradigm Identified**: Using libraries = weakness, NIH (Not Invented Here) syndrome

**Transcendence**: Recognize paradigm itself is cultural artifact, not truth. Could value creation actually come from:
- Assembling best components intelligently
- Focus on domain-specific problems
- Standing on shoulders of giants

**Impact**: 10× productivity improvement by questioning paradigm

### Detection Questions
- What assumptions are so fundamental no one questions them?
- What would be "unthinkable" to leadership?
- What beliefs persist despite contradictory evidence?
- What would someone from different culture/industry find bizarre?

## L2: Paradigm Shift

**Definition**: Change the fundamental mental model that drives system structure and goals.

### Manufacturing Quality Example
**Old Paradigm**: "Quality is QA's job, engineering designs, QA catches mistakes"

**New Paradigm**: "Quality is everyone's job, errors caught early cost 100× less than errors caught late"

**Intervention**:
1. Calculate cost: Customer defect = $50K, QA defect = $500, Engineer self-caught = $50
2. Share stories: "$2M satellite failed because engineer didn't check units, felt 'not my job'"
3. Rename QA to "Quality Engineering", integrate into design reviews
4. Celebrate engineers who catch their own mistakes

**Timeline**: 3-6 months for culture shift, 12 months for full adoption

**Metrics**: % defects caught by engineers (should increase), total defects (should decrease)

### Learning System Example
**Old Paradigm**: "Learning = consuming theory, then applying"

**New Paradigm**: "Learning = solving problems, theory pulled as needed"

**Intervention**:
1. Show data: Problem-first learners retain 3× more than theory-first
2. Pilot experiment: Two cohorts, different approaches, measure outcomes
3. Success stories: "I solved 3 real problems vs read 10 papers"

**Timeline**: Immediate for individuals, 3-12 months for institutions

### Defense Contracting Example
**Old Paradigm**: "Contract negotiation is zero-sum, win-lose game"

**New Paradigm**: "Value creation through fair risk allocation based on control"

**Intervention**:
1. Show case study: Fair contract = 40% fewer disputes, 25% faster execution
2. Calculate cost of disputes vs cost of upfront structuring
3. Demonstrate "expanding pie" through incentive alignment

**Timeline**: 6-12 months for cultural shift in organization

### Resistance Patterns
- **Authority bias**: "Industry has always done it this way"
- **Sunk cost**: "We've invested in current paradigm"
- **Identity threat**: "If I change paradigm, I was wrong before"
- **Social proof**: "No one else does it differently"

### Success Indicators
- Language changes ("we" vs "them" between departments)
- New questions asked in meetings
- Proposals include options that would have been "unthinkable" before
- Reduced resistance to changes aligned with new paradigm

## L3: System Goals

**Definition**: The purpose the system is designed to achieve (not stated goals, actual operational goals).

### Misaligned Goals - Defense Production Example
**Stated Goal**: "Deliver high-quality defense products on time, on budget"

**Real Goal** (revealed by incentives):
- Production manager: Maximize units shipped this month (bonus based on volume)
- Quality manager: Minimize defects detected (avoid looking bad)
- Engineering manager: Minimize rework hours (efficiency metric)

**Result**: Everyone locally optimizing, system globally degrading

**Intervention**:
1. **Define true system goal**: "Maximize customer capability delivered per dollar spent" (not units, not perfection, but value)
2. **Cascade metrics**:
   - Production: First-pass yield (not volume)
   - Quality: Prevention effectiveness (not detection effectiveness)
   - Engineering: Design reliability (not hours spent)
3. **Align incentives**: Team bonuses based on system goal, not departmental metrics

**Expected Impact**: 30-50% improvement in overall throughput when everyone optimizes for same goal

### Wrong Goal - Software Velocity Example
**Stated Goal**: "Ship features to customers"

**Real Goal** (revealed by metrics): "Maximize story points per sprint"

**Problems**:
- Story points gameable (inflate estimates)
- Features shipped ≠ features used
- Quality unmeasured = quality ignored
- Tech debt accumulates (no metric for it)

**Intervention - Redefine Goal**: "Maximize customer value per dollar invested"

**New Metrics**:
- Deploy frequency × Stability (not just frequency)
- Feature adoption rate (% of features actually used)
- Code health score (tech debt, complexity)
- Customer satisfaction trend

**Impact**: Shifts from "busy work" to "valuable work"

### Learning System Example
**Stated Goal**: "Master systems thinking"

**Real Goal** (revealed by behavior): "Accumulate theory knowledge"

**Problem**: Optimizing for input (reading hours) not output (problem-solving capability)

**Intervention - Redefine Goal**: "Solve 3 real engineering problems using systems thinking"

**New Metrics**:
- Problems solved (not papers read)
- Tools built and validated (not just built)
- Concepts applied (not just understood)

**Impact**: Learning becomes purposeful, measurable, valuable

### Detection Questions
- What do performance reviews actually reward?
- What gets discussed in leadership meetings?
- What causes someone to get promoted/fired?
- If you had to choose between stated goal and real goal, which would you pick?
- What metrics drive daily decisions?

### Common Goal Misalignments
| Stated Goal | Actual Goal (Revealed) | Leverage Point |
|-------------|------------------------|----------------|
| Customer satisfaction | Call center call volume | Redefine: Customer issues resolved |
| Innovation | Patent count | Redefine: Commercial impact of innovations |
| Employee development | Training hours | Redefine: Skill application in real work |
| System security | Compliance checkbox | Redefine: Actual threat resistance |
| Quality products | Defects caught by QA | Redefine: Defects prevented in design |

### Implementation Checklist
- [ ] Identify stated vs actual goals (watch behavior, not claims)
- [ ] Calculate cost of goal misalignment (disputes, rework, waste)
- [ ] Define measurable system goal aligned with true value creation
- [ ] Cascade goal to all subsystems (ensure coherence)
- [ ] Change incentives to align with new goal
- [ ] Make goal visible (dashboards, meetings, reviews)
- [ ] Track leading indicators of goal shift (language, proposals, decisions)
- [ ] Iterate every quarter based on unintended consequences
