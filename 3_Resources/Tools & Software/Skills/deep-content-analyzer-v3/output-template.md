# Output Template: 9-Part Analysis Document

## File Naming
`[Content_Title]_Full_Analysis.md`

Save to `/mnt/user-data/outputs/` and use `present_files`.

---

## Document Structure

```markdown
# Multi-Framework Analysis: "[Content Title]"
## Systems Thinking + Meta-Learning + First-Principles Debate

**Source:** [Author/source]
**Date:** [Date]
**Frameworks Applied:** [List]

---

## PART 1: CLARIFICATION — What Is Actually Being Said?

### 1.1 Core Thesis (Feynman Simplification)
60-second explanation in plain language.
Everyday analogy (kitchen, sports, construction — visceral and memorable).
What the author gets right and what they miss.

### 1.2 Taxonomy / Chunked Breakdown
Content decomposed into structured categories.
Table showing: Chunk | Key Concepts | Prerequisites | Est. Time

---

## PART 2: SYSTEMS THINKING ANALYSIS

### 2.1 Stock-Flow Map
Each stock with: Name, Current Level, Type (Buffer/Constraint),
Inflows (name, rate, control, delay), Outflows (same), Pattern.
Critical Stocks Ranking table.

### 2.2 Feedback Loop Detection
Loop inventory table: Loop | Name | Structure | Speed | Dominance | Status
Dominance narrative explaining which loop currently drives behavior.

### 2.3 System Archetype Detection
Primary archetype with: pattern description, evidence, confidence, counter-strategy.
Secondary archetypes if present.

### 2.4 Meadows' Leverage Point Analysis
Leverage point table: L[X] | Name | Priority | Evidence | Intervention | Impact | Timeline
Phase 1-3 Intervention Cascade.

---

## PART 3: META-LEARNING ANALYSIS

### 3.1 Chunking Breakdown — Learning Architecture
Dependency hierarchy (often differs from presentation order — note where).
Optimal learning sequence with time estimates.

### 3.2 Feynman Test — Diagnostic Questions
3 questions at increasing depth:
- Level 1 (Comprehension): Can you explain X?
- Level 2 (Application): How would you apply X in situation Y?
- Level 3 (System): What happens to the whole system when X changes?

### 3.3 Mnemonic
1-2 acronyms. For each: the word, what each letter means, retrieval instructions.

### 3.4 Self-Assessment Rubric
Table: Dimension | 1 (Novice) | 3 (Competent) | 5 (Expert)
Behavioral indicators at each level.
Scoring bands with interpretation.

### 3.5 Interleaving Schedule
Weekly table: Week | Morning Block | Afternoon Block | Practice Focus
No two consecutive blocks on same primary topic.

### 3.6 Targeted Drills
2-5 exercises. Each: purpose, duration, instructions, progression, scoring.

### 3.7 Learning Journal Prompts
5-7 reflection questions. At least one about feedback loops, one about meta-learning.

---

## PART 4: CONCEPT EVALUATION — Right, Wrong, and Missing

### 4.1 Design Review: Strengths
What the author gets fundamentally right, with systems thinking evidence.
Be specific — "the structured delegation example is valuable because it's an L5 intervention."

### 4.2 Design Review: Weaknesses  
What's wrong or incomplete, with specific alternatives.
Be structural — not "should have more detail" but "treats X as static when it's dynamic."

### 4.3 What's Missing: Hidden Dimensions
Dimensions the author ignores that systems analysis reveals.
Each with: what's missing, why it matters, what changes if you include it.

---

## PART 5: FIRST-PRINCIPLES DEBATE

### 5.1 Debate Points
3-7 points. Each: claim, counter-argument, corrected frame, systems integration.

### 5.2 Current vs. Fundamental Limitation Analysis
Table: Limitation Cited | Type (Current/Fundamental) | Evidence | Implication

### 5.3 Rate-of-Change Analysis
Key rates identified. Survival/success condition.
Boundary movement analysis if applicable.

---

## PART 6: ARCHITECT EXPANDED FRAMEWORK

### 6.1 First-Principles Reduction
Original → Irreducible elements → Why reduction matters.
ASCII layered architecture diagram.

### 6.2 Framework Mnemonic: [WORD]
Each letter = principle. Retrieval instructions.

### 6.3 Complete System Map
ASCII diagram showing all stocks, layers, and flows.

### 6.4 Extended Feedback Loop Inventory
Complete R + B table including debate-discovered loops.

### 6.5 System Archetypes
Table with counter-strategies.

### 6.6 Leverage Point Cascade
Phase 1-3 intervention tables incorporating debate insights.

### 6.7 Extended Self-Assessment Rubric
All framework dimensions. Scoring bands.

### 6.8 Targeted Drills
3-7 drills. Each tied to a loop or leverage point.

### 6.9 Extended Interleaving Schedule
12-16 weeks mixing original + expanded content.

### 6.10 Focus Session Design
90-min block structure.

### 6.11 Learning Journal Template
Keyed to framework dimensions.

---

## PART 7: USE CASE RECOMMENDATIONS

### 7.1 Audience Map
Table: Audience | Focus | Supplement | Priority Action

### 7.2 Implementation Roadmap  
Phase 1-3: actions, timelines, deliverables, metrics.

---

## PART 8: INTEGRATED SYNTHESIS

### 8.1 The Single Most Important Insight
One paragraph. The deepest finding from the entire analysis.

### 8.2 System Archetype Warning
The most dangerous dynamic. Why it's dangerous. How to counter it.

### 8.3 Counter-Intuitive Insights
3-5 numbered insights that would surprise even careful readers.

### 8.4 The Three Laws
3 irreducible principles. Each: name, one-sentence statement, 2-3 sentences on why.

---

## PART 9: PROGRESS TRACKING

### 9.1 Competency Assessment Grid
Evidence-based progression table.

### 9.2 Weekly Review Questions
5 questions for ongoing self-assessment.

---

*Analysis produced using: [list all frameworks applied]*
```

## Quality Gates (Quick Reference)

| Area | Gate | Why It Matters |
|------|------|---------------|
| Stocks | ≥4 with full details | Fewer = superficial system understanding |
| Loops | ≥3 classified R/B | Loops ARE the system dynamics |
| Archetypes | ≥1 with evidence | Archetypes explain WHY things keep happening |
| Leverage | ≥5 across levels | All-L12 = low-leverage thinking |
| Debate | ≥3 formal points | Fewer = not challenging the content enough |
| C vs. F | Applied to ≥2 claims | The most valuable analytical distinction |
| Rate analysis | Included | Without rates, it's a snapshot not dynamics |
| Missing dims | ≥3 identified | The author's blind spots are your best findings |
| Mnemonic | Meaningful word | Random letters don't stick |
| Three Laws | Genuinely different | Restatements = haven't found the structure |
| Word count | >5,000 | Deep analysis requires depth |
