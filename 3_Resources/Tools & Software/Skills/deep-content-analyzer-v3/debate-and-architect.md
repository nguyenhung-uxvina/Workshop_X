## 1. The 6-Step Debate Process

### Step 1: State Claims Precisely
Strip rhetoric, metaphor, and persuasive framing. What is the author actually asserting as true? Write it as a single declarative sentence.

### Step 2: Decompose to Fundamentals
Ask: "What would have to be true for this claim to hold?"

Classify each assumption:
- **Axiomatic** — Self-evident, definitionally true
- **Empirical** — Testable, currently supported by evidence
- **Conventional** — Widely believed but not necessarily true from first principles
- **Speculative** — Projection without strong evidence

Most interesting claims rest on Conventional assumptions dressed as Empirical ones.

### Step 3: Current vs. Fundamental Limitation Test

For any limitation the author cites:

**Current limitations (will eventually be overcome):**
- Technology gap but trajectory is clear
- Scaling/cost barriers that reduce over time  
- Regulatory gaps that will be filled
- Context window / reasoning depth limits
→ Don't bet strategy on these lasting

**Fundamental limitations (physics-level constraints):**
- Requires genuine embodiment in physical world
- Requires consciousness or subjective experience
- Requires skin-in-the-game (actual consequences for the entity)
- Violates information-theoretic limits
- Requires 200,000 years of evolved social cognition hardware
→ Safe to build strategy on these lasting

### Step 4: Rate-of-Change Analysis

Identify variables the author treats as static but are actually dynamic:

```
Key rates to identify:
  dH/dt = Human capability change rate (learning speed)
  dA/dt = Agent/technology capability expansion rate
  dR/dt = Regulatory/institutional change rate
  dM/dt = Market/competitive evolution rate

Survival/success condition: Define as relationship between rates
  Example: dH/dt > dA/dt in your domain = career safety
  Example: dM/dt > dR/dt = regulatory arbitrage window
```

Map boundary movement if the content discusses a boundary between human/machine tasks:
```
[Year]: What agents handle → What humans handle
[Year+1]: Boundary has moved → New human territory
[Year+2]: Boundary moves further → Even newer territory

Question: Is the boundary moving faster or slower than human adaptation?
```

### Step 5: Missing Dimensions Checklist

Systematically check what the author ignores:

| Dimension | What to Check | Why It Matters |
|-----------|--------------|----------------|
| **Taste** | Correct vs. excellent distinction? | When baseline quality rises, only taste differentiates |
| **Skin in the Game** | Entities with vs. without consequences? | Consequenceless entities recommend differently |
| **Compound Stack** | Skills as independent vs. multiplicative? | Unique combinations create exponential value |
| **Trust Capital** | Trust as explicit asset? | Builds linearly, collapses exponentially |
| **Physical-World Interface** | Embodiment requirements? | Infinite state space, irreversible consequences |
| **Power Dynamics** | Who controls, who benefits, who's blamed? | Change is political, not just technical |
| **Information Asymmetry** | Who has context vs. who has data? | Contextual arbitrage creates outsized value |
| **Regulatory Dynamics** | Regulatory vacuum/window? | 2-5 year arbitrage windows are enormously valuable |
| **Co-Evolution** | Static or co-evolutionary relationship? | Each improvement creates new problems |
| **Second-Order Effects** | What happens AFTER the solution? | Most authors stop at first-order |
| **Temporal Dynamics** | Snapshot vs. trajectory? | The direction matters more than the position |
| **Selection Bias** | Only supporting evidence? | Authors naturally over-represent confirmatory data |

### Step 6: Generate Debate Points

For each significant gap, produce:

```markdown
## Debate Point N: [One-sentence title capturing the counter-argument]

**The article's claim:** [Precise restatement]

**First-principles counter-argument:**
[2-4 paragraphs: WHY the claim is incomplete from fundamentals. 
Use physics analogies, rate analysis, or information theory.]

**The corrected frame:**
[The more accurate version — what the author should have said]

**Systems Thinking integration:**
[How this connects to stocks/flows/loops/leverage identified in Phase 3]
```

Target: 3-7 debate points. At least one should address rate-of-change, and at least one should surface a missing dimension.

## 2. The 11-Step ARCHITECT Generation Process

### Step 1: Identify Irreducible Elements

Collapse the author's list into the minimum set that can't be reduced further:

```
ORIGINAL (author's model):
  [List their items — typically 5-12 in a flat list]

FIRST-PRINCIPLES REDUCTION:
  [Group into 3±1 irreducible elements with hierarchy]

WHY THE REDUCTION MATTERS:
  [What hidden structure the flat list obscured]
```

### Step 2: Design Layered Architecture

```
┌──────────────────────────────────┐
│  LAYER N: [Highest capability]   │
├──────────────────────────────────┤
│  LAYER N-1: [Supporting]         │
├──────────────────────────────────┤
│  LAYER 1: [Foundation]           │
└──────────────────────────────────┘
     ↑ Substrate: [META-CAPABILITY]
OUTPUT: [What the system produces]
```

### Step 3: Create Framework Mnemonic

Requirements:
- A meaningful word (not random letters) connected to the domain
- Each letter maps to a specific, actionable principle
- Retrievable: you should be able to recall what each letter means without the document
- Include retrieval instructions alongside the mnemonic

### Step 4: Map Complete System Dynamics

Create a full loop inventory table that includes BOTH Phase 3 loops AND new loops discovered during the debate:

```
| Loop | Name | Structure | Speed | Dominance | Status |
|------|------|-----------|-------|-----------|--------|
| R1   | [Descriptive] | [A → B → C → A] | Fast/Med/Slow | HIGH/MED/LOW | Active/Dormant |
```

### Step 5: Map System Archetypes

```
| Archetype | Where | Danger | Counter-Strategy |
|-----------|-------|--------|-----------------|
```

### Step 6: Generate Leverage Point Cascade

Expand Phase 3 leverage points with debate insights:

**Phase 1 (Week 1-4): Information + Rules**
| # | Intervention | Leverage | Impact | Pilot Action |
|---|-------------|----------|--------|-------------|

**Phase 2 (Week 5-12): Loop Modification**
[Same table format]

**Phase 3 (Month 4-6): Goals + Architecture**
[Same table format]

### Step 7: Build Extended Rubric

Cover all framework dimensions:
```
| Dimension | 1 (Vulnerable) | 3 (Building) | 5 (Antifragile) |
|-----------|----------------|---------------|-----------------|
| [Each mnemonic letter] | [Behavioral] | [Behavioral] | [Behavioral] |
```

Scoring bands:
- Low range: EXPOSED — immediate action needed
- Mid range: TRANSITIONING — focus on weak dimensions
- High range: POSITIONED — compound and teach
- Top range: ANTIFRAGILE — environment improvement makes you stronger

### Step 8: Design Targeted Drills

3-7 drills, each with:

```
**Drill N: "[Name]" ([Frequency], [Duration])**
Purpose: [Which framework dimension]
Loop/Leverage: [Which R/B loop or L-point this addresses]

Instructions:
1. [Step]
2. [Step]
3. [Step]

Progression:
- Week 1-2: [Beginner]
- Week 3-4: [Intermediate]
- Week 5+: [Advanced]

Scoring: [How to know you're improving]
```

### Step 9: Create Interleaving Schedule

12-16 weeks mixing original content + expanded framework:

```
| Week | Morning (90 min) | Afternoon (60 min) | Drill |
|------|------------------|--------------------|-------|
```

No two consecutive blocks on the same primary topic.

### Step 10: Design Focus Sessions

90-minute optimal block:

```
0-5:    Intention ("Which dimension am I building?")
5-25:   Deep Work Block 1 (primary skill)
25-27:  Micro-break
27-47:  Deep Work Block 2 (application)
47-49:  Micro-break
49-69:  Deep Work Block 3 (integration/compound stack)
69-75:  Reflection (journal prompts)
75-80:  Planning (tomorrow's priority)
80-90:  Buffer
```

### Step 11: Distill Three Laws

The compression test. Three principles that are:
- **Memorable** — Can be recalled without the document
- **Generative** — Unpacking any one law reconstructs a large chunk of the framework
- **Non-obvious** — Surprised even you during the analysis

Format:
```
### Law 1: The [Name] Law
[One sentence]
[2-3 sentences: why this is a law, not just a guideline]
```

Past analyses consistently produce laws in three categories:
1. A paradigm shift (how you see the system)
2. A rate or compound dynamic (what accumulates matters more than what happens)
3. A structural counter-intuitive insight (the opposite of what most people assume)
